NETWORK OSI LAYERS BY TOM7

[ 7-LAYER HOLOGRAPHIC DATABASE SYSTEM BY TOM7 ]

01

LAYER 1 - PHYSICAL

Ethernet Cable

Wired Physical Medium

Kabel Ethernet adalah media fisik yang digunakan untuk menghubungkan perangkat dalam jaringan. Menggunakan twisted pair copper wires untuk mentransmisikan sinyal listrik yang merepresentasikan data digital.

Tipe: Cat5e, Cat6, Cat6a, Cat7, Cat8
Kecepatan: 100 Mbps - 40 Gbps
Jarak: Hingga 100 meter
Standar: IEEE 802.3

Fiber Optic

High-Speed Optical Medium

Fiber optik menggunakan serat kaca atau plastik untuk mentransmisikan data dalam bentuk pulsa cahaya. Menawarkan bandwidth sangat tinggi dan imun terhadap interferensi elektromagnetik.

Tipe: Single-mode, Multi-mode
Kecepatan: Hingga 100 Gbps+
Jarak: Hingga 100+ km
Keunggulan: Bandwidth tinggi, jarak jauh

Wireless Radio Waves

Wireless Physical Medium

Gelombang radio digunakan untuk transmisi data nirkabel. Menggunakan spektrum frekuensi radio untuk mengirimkan sinyal data antar perangkat tanpa kabel fisik.

Frekuensi: 2.4 GHz, 5 GHz, 6 GHz
Standar: 802.11a/b/g/n/ac/ax
Jarak: 30-100 meter indoor
Kecepatan: 11 Mbps - 9.6 Gbps

Network Interface Card (NIC)

Hardware Interface

NIC adalah hardware yang memungkinkan komputer terhubung ke jaringan. Bertanggung jawab mengkonversi data digital menjadi sinyal yang dapat ditransmisikan melalui media fisik.

Tipe: Ethernet, Wireless, Fiber
Interface: PCI, PCIe, USB
Kecepatan: 10/100/1000 Mbps, 10 Gbps
Identifier: MAC Address

Repeater

Signal Amplifier

Repeater adalah perangkat yang menerima sinyal, memperkuatnya, dan mentransmisikan kembali untuk memperpanjang jarak transmisi. Bekerja di layer physical untuk regenerasi sinyal.

Fungsi: Amplifikasi sinyal
Kegunaan: Perpanjangan jarak
OSI Layer: Layer 1
Operasi: Regenerasi sinyal

Hub

Multi-port Repeater

Hub adalah perangkat yang menghubungkan beberapa komputer dalam jaringan. Bekerja dengan menyiarkan data yang diterima ke semua port, tanpa filtering atau routing.

Tipe: Passive, Active, Intelligent
Operasi: Broadcast ke semua port
Port: 4, 8, 16, 24 port
Collision: Collision domain shared
02

LAYER 2 - DATA LINK

Switch

Intelligent Network Device

Switch adalah perangkat jaringan yang beroperasi di layer 2 OSI. Menggunakan MAC address untuk meneruskan frame data hanya ke port tujuan yang tepat, meningkatkan efisiensi dan keamanan jaringan.

Tipe: Managed, Unmanaged, Smart
MAC Table: MAC address learning
Speed: Gigabit, 10 Gigabit
VLAN: Virtual LAN support

Bridge

Network Segment Connector

Bridge menghubungkan dua atau lebih segmen jaringan menjadi satu jaringan logis. Memfilter traffic berdasarkan MAC address untuk mengurangi collision domain.

Fungsi: Segmentasi jaringan
Filtering: MAC-based filtering
Collision: Memisahkan collision domain
Layer: Data Link Layer

MAC Address

Physical Address

MAC (Media Access Control) Address adalah alamat hardware unik yang diassign ke setiap network interface card. Terdiri dari 48-bit (6 bytes) yang ditulis dalam format hexadecimal.

Format: XX:XX:XX:XX:XX:XX
Uniqueness: Globally unique
OUI: Vendor identification
Keamanan: MAC filtering

VLAN (Virtual LAN)

Logical Network Segmentation

VLAN adalah teknologi yang memungkinkan segmentasi jaringan secara logis pada level switch. Memungkinkan pemisahan broadcast domain tanpa perlu hardware tambahan.

Standar: IEEE 802.1Q
Segmentasi: Logical separation
Keamanan: Traffic isolation
VLAN ID: 1-4094

STP (Spanning Tree Protocol)

Loop Prevention Protocol

STP adalah protokol yang mencegah terjadinya switching loop dalam jaringan. Secara otomatis mendeteksi dan menonaktifkan jalur redundan untuk menghindari broadcast storm.

Standar: IEEE 802.1D
Fungsi: Loop prevention
Variants: RSTP, MSTP
Root Bridge: Election algorithm

Frame

Data Link Layer PDU

Frame adalah Protocol Data Unit (PDU) pada layer 2. Mengenkapsulasi packet dari layer 3 dengan menambahkan header (MAC source & destination) dan trailer (FCS untuk error checking).

Komponen: Header, Payload, Trailer
Size: 64 - 1518 bytes
FCS: Error detection
Encapsulation: Layer 3 packets
03

LAYER 3 - NETWORK

Router

Network Gateway Device

Router adalah perangkat yang menghubungkan berbagai jaringan dan menentukan jalur terbaik untuk pengiriman data. Bekerja di layer 3 menggunakan IP address untuk routing decisions.

Fungsi: Inter-network routing
Routing: Static, Dynamic
Firewall: Built-in security
NAT: Address translation

IP Address

Logical Network Address

IP Address adalah alamat logis yang digunakan untuk mengidentifikasi perangkat dalam jaringan. Tersedia dalam dua versi: IPv4 (32-bit) dan IPv6 (128-bit).

IPv4: 192.168.1.1 (32-bit)
IPv6: 2001:db8::1 (128-bit)
Classes: A, B, C, D, E
Private: 10.x, 172.16.x, 192.168.x

ICMP

Internet Control Message Protocol

ICMP digunakan untuk mengirim pesan error dan informasi operasional tentang jaringan. Paling sering digunakan melalui utility ping dan traceroute untuk network troubleshooting.

Protocol: Network layer
Ping: Echo request/reply
Traceroute: Path discovery
Messages: Error reporting

Routing Protocols

Dynamic Path Selection

Routing protocols memungkinkan router untuk saling berbagi informasi routing dan menentukan jalur terbaik secara dinamis. Termasuk RIP, OSPF, EIGRP, dan BGP.

RIP: Distance vector, hop count
OSPF: Link state, area-based
BGP: Path vector, Internet routing
Updates: Automatic convergence

Packet

Network Layer PDU

Packet adalah unit data pada network layer yang berisi IP header dan payload dari layer transport. Header mencakup source dan destination IP address serta informasi routing.

Header: Source/Dest IP, TTL, Protocol
TTL: Time To Live
Fragmentation: MTU handling
Checksum: Header integrity

Layer 3 Switch

Multilayer Switch

Layer 3 Switch menggabungkan fungsi switch dan router dalam satu perangkat. Dapat melakukan switching di layer 2 dan routing di layer 3 dengan hardware acceleration.

Fungsi: Switching + Routing
Speed: Wire-speed routing
VLAN: Inter-VLAN routing
ASIC: Hardware-based
04

LAYER 4 - TRANSPORT

TCP

Transmission Control Protocol

TCP adalah protokol transport yang connection-oriented dan reliable. Menjamin pengiriman data yang terurut dan bebas error melalui mekanisme acknowledgment dan retransmission.

Type: Connection-oriented
Reliable: Error checking & recovery
Flow Control: Windowing
Ports: 0-65535

UDP

User Datagram Protocol

UDP adalah protokol transport connectionless yang tidak menjamin pengiriman data. Lebih cepat daripada TCP karena tidak ada overhead untuk connection establishment dan error checking.

Type: Connectionless
Speed: Fast, low latency
Unreliable: Best-effort delivery
Use Case: Streaming, gaming, VoIP

Port Numbers

Service Identifiers

Port numbers digunakan untuk mengidentifikasi aplikasi atau service spesifik yang berjalan pada host. Range: 0-65535, dibagi menjadi well-known (0-1023), registered (1024-49151), dan dynamic (49152-65535).

HTTP: Port 80
HTTPS: Port 443
FTP: Port 21
SSH: Port 22

Segment

Transport Layer PDU

Segment adalah unit data pada transport layer (TCP). Berisi TCP/UDP header dengan port numbers, sequence numbers, dan data dari application layer.

Header: Source/Dest ports
Sequence: Ordered delivery (TCP)
ACK: Acknowledgment (TCP)
Window: Flow control

Three-Way Handshake

TCP Connection Establishment

Three-way handshake adalah proses yang digunakan TCP untuk membuat koneksi. Melibatkan exchange SYN, SYN-ACK, dan ACK packets untuk establish reliable connection.

Step 1: Client → SYN
Step 2: Server → SYN-ACK
Step 3: Client → ACK
Result: Connection established

Load Balancer

Traffic Distribution

Load Balancer mendistribusikan traffic jaringan ke multiple servers untuk meningkatkan reliability, performance, dan scalability aplikasi.

Layer: Layer 4 (Transport)
Algorithm: Round-robin, least connections
Health Check: Server monitoring
Scalability: Horizontal scaling
05

LAYER 5 - SESSION

NetBIOS

Network Basic Input/Output System

NetBIOS menyediakan services untuk session layer yang memungkinkan aplikasi pada komputer berbeda untuk berkomunikasi dalam LAN. Digunakan untuk name resolution dan session management.

Services: Name, Session, Datagram
Ports: 137, 138, 139
Names: 15-character limit
Platform: Windows networking

RPC

Remote Procedure Call

RPC memungkinkan program untuk execute procedure di komputer remote seolah-olah adalah local procedure call. Menyembunyikan kompleksitas network communication dari developer.

Type: Request-response protocol
Implementation: gRPC, XML-RPC
Use Case: Distributed systems
Communication: Synchronous

Session Token

Session Authentication

Session token adalah identifier unik yang digunakan untuk maintain state antara client dan server dalam stateless protocols seperti HTTP. Digunakan untuk authentication dan authorization.

Type: Cookie-based, Token-based
Security: Encrypted, signed
Expiration: Time-limited
Format: JWT, Session ID

PPTP

Point-to-Point Tunneling Protocol

PPTP adalah protokol yang digunakan untuk implement VPN. Beroperasi di session layer untuk membuat secure tunnel antara client dan server melalui Internet.

Port: TCP 1723
Security: MPPE encryption
Protocol: GRE tunneling
Speed: Fast, low overhead

Sockets API

Programming Interface

Sockets API menyediakan interface untuk aplikasi berkomunikasi melalui jaringan. Abstraksi yang memungkinkan programmer membuat network applications tanpa menangani low-level details.

Types: Stream (TCP), Datagram (UDP)
Functions: bind(), listen(), accept()
Layer: Transport + Session
Platform: Cross-platform

SIP

Session Initiation Protocol

SIP adalah signaling protocol untuk initiating, maintaining, dan terminating real-time sessions untuk voice, video, dan messaging applications.

Port: 5060 (UDP/TCP), 5061 (TLS)
Use Case: VoIP, video calls
Messages: INVITE, ACK, BYE
Architecture: Client-server
06

LAYER 6 - PRESENTATION

SSL/TLS

Encryption Protocol

SSL (Secure Sockets Layer) dan TLS (Transport Layer Security) adalah cryptographic protocols yang menyediakan secure communication melalui network. TLS adalah successor dari SSL.

Version: TLS 1.2, TLS 1.3
Encryption: AES, ChaCha20
Certificate: X.509
Handshake: Key exchange

Data Encoding

Format Conversion

Presentation layer bertanggung jawab untuk encoding dan decoding data. Mengkonversi data dari application format ke network format dan sebaliknya.

Formats: ASCII, EBCDIC, Unicode
Compression: GZIP, DEFLATE
Serialization: JSON, XML, Protocol Buffers
Character Sets: UTF-8, UTF-16

Encryption/Decryption

Data Security

Presentation layer handle encryption dan decryption untuk mengamankan data transmission. Menggunakan cryptographic algorithms untuk protect data confidentiality dan integrity.

Symmetric: AES, DES, 3DES
Asymmetric: RSA, ECC
Hash: SHA-256, MD5
Key Exchange: Diffie-Hellman

Data Compression

Bandwidth Optimization

Compression mengurangi size data yang ditransmisikan melalui network. Meningkatkan efficiency bandwidth dan mempercepat transfer speed tanpa data loss.

Lossless: ZIP, GZIP, LZ77
Lossy: JPEG, MP3, H.264
Ratio: 50-90% reduction
Speed: Real-time compression

MIME

Multipurpose Internet Mail Extensions

MIME adalah standard untuk format email messages dan HTTP content. Mendefinisikan content types dan encoding untuk berbagai jenis file dan data.

Types: text/html, image/png, video/mp4
Email: Attachment support
HTTP: Content-Type header
Encoding: Base64, Quoted-printable

Media Formats

Image/Video/Audio Presentation

Presentation layer menangani format dan conversion berbagai media types. Memastikan data dapat dipresentasikan dengan benar di receiving end.

Image: JPEG, PNG, GIF, WebP
Video: MP4, AVI, MKV, WebM
Audio: MP3, AAC, FLAC, OGG
Conversion: Format transformation
07

LAYER 7 - APPLICATION

HTTP/HTTPS

Web Protocol

HTTP (Hypertext Transfer Protocol) adalah protokol dasar World Wide Web. HTTPS menambahkan encryption layer menggunakan SSL/TLS untuk secure web browsing.

Port: HTTP 80, HTTPS 443
Methods: GET, POST, PUT, DELETE
Status: 200 OK, 404 Not Found
Security: TLS encryption (HTTPS)

SMTP/POP3/IMAP

Email Protocols

SMTP (Simple Mail Transfer Protocol) untuk sending email, POP3 (Post Office Protocol) dan IMAP (Internet Message Access Protocol) untuk receiving email dari server.

SMTP: Port 25, 587 (secure)
POP3: Port 110, 995 (secure)
IMAP: Port 143, 993 (secure)
IMAP: Server synchronization

FTP/SFTP

File Transfer Protocol

FTP digunakan untuk transfer files antara client dan server. SFTP (SSH File Transfer Protocol) adalah secure version yang mengenkripsi file transfers.

FTP: Port 20 (data), 21 (control)
SFTP: Port 22 (SSH)
Security: SFTP encrypted
Mode: Active, Passive

DNS

Domain Name System

DNS mentranslasikan domain names (www.example.com) ke IP addresses. Bekerja seperti phonebook Internet yang memudahkan manusia mengakses website.

Port: 53 (UDP/TCP)
Records: A, AAAA, CNAME, MX, TXT
Hierarchy: Root, TLD, Authoritative
Caching: TTL-based

DHCP

Dynamic Host Configuration Protocol

DHCP secara otomatis mengassign IP addresses dan network configuration ke devices di jaringan. Eliminates manual configuration dan memudahkan network management.

Port: 67 (server), 68 (client)
Process: DORA (Discover, Offer, Request, ACK)
Lease: Time-limited assignment
Config: IP, Subnet, Gateway, DNS

SSH

Secure Shell

SSH menyediakan secure channel untuk remote command-line access ke servers. Mengenkripsi semua communication termasuk passwords dan commands.

Port: 22
Encryption: End-to-end encrypted
Auth: Password, Public key
Features: Port forwarding, tunneling

Telnet

Remote Terminal Protocol

Telnet adalah protokol untuk remote terminal access. TIDAK secure karena mengirim data termasuk passwords dalam plaintext. Sudah largely replaced oleh SSH.

Port: 23
Security: Unencrypted
Use: Remote CLI access
Status: Deprecated for SSH

SNMP

Simple Network Management Protocol

SNMP digunakan untuk monitoring dan managing network devices. Collect data tentang device status, performance metrics, dan network health.

Port: 161 (agent), 162 (trap)
Version: SNMPv1, v2c, v3
MIB: Management Information Base
Traps: Alerts & notifications

NTP

Network Time Protocol

NTP synchronize clocks pada komputer dalam network. Penting untuk logging, security certificates, dan distributed systems yang memerlukan time consistency.

Port: 123 (UDP)
Stratum: Hierarchical time sources
Accuracy: Millisecond precision
Source: Atomic clocks, GPS

SMB/CIFS

File Sharing Protocol

SMB (Server Message Block) dan CIFS (Common Internet File System) digunakan untuk sharing files, printers, dan resources dalam network, terutama Windows networks.

Port: 445 (SMB), 139 (NetBIOS)
Platform: Windows, Linux (Samba)
Security: SMB3 encryption
Features: File & printer sharing

LDAP

Lightweight Directory Access Protocol

LDAP adalah protokol untuk accessing dan maintaining distributed directory information services. Commonly used untuk user authentication dan organizational data.

Port: 389 (LDAP), 636 (LDAPS)
Structure: Hierarchical tree
Use Case: Active Directory, OpenLDAP
Auth: Single sign-on (SSO)

REST API

Web Service Architecture

REST (Representational State Transfer) adalah architectural style untuk designing networked applications. Menggunakan HTTP methods untuk CRUD operations pada resources.

Protocol: HTTP/HTTPS
Format: JSON, XML
Methods: GET, POST, PUT, DELETE
Stateless: No session state
